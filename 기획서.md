# 쿠키런 클래식 스타일 러너 게임 — 개발 기획서

> 클로드코드(Claude Code)에 전달하기 위한 기획서입니다.
> **핵심 원칙: 한 번에 전부 만들지 말고, 아래 Phase 순서대로 하나씩 구현하고 테스트한 뒤 다음 단계로 넘어갈 것.**

---

## 1. 프로젝트 개요

- **장르**: 가로 스크롤 엔드리스 러너 (쿠키런 클래식 / 오븐브레이크 스타일)
- **핵심 재미**: 캐릭터가 자동으로 달리고, 플레이어는 **점프·슬라이드**만으로 장애물을 피하고 아이템(젤리)을 먹어 점수를 쌓는다.
- **목표**: 최대한 오래 살아남으며 높은 점수 획득.
- **플랫폼**: 웹 브라우저 (PC 키보드 + 모바일 터치 모두 지원), HTML 단일 파일.

---

## 2. 개발 방식 (가장 중요)

1. **MVP(Phase 1)를 먼저 완성**하고 브라우저에서 실제로 플레이되는지 확인한다.
2. 각 Phase는 **이전 Phase가 정상 동작하는 상태**에서만 다음으로 진행한다.
3. 각 Phase 끝에 있는 **테스트 체크리스트**를 통과해야 다음 단계로 넘어간다.
4. 모든 수치(속도, 점프력, 중력 등)는 **파일 상단의 CONFIG 객체로 모아** 나중에 쉽게 조정할 수 있게 한다.
5. 그래픽은 처음엔 **단순 도형/색상/이모지**로 대체하고, 게임이 돌아간 다음에 다듬는다.

---

## 3. 기술 스택

- **HTML5 + Canvas + 순수 JavaScript (Vanilla JS)**
- 외부 라이브러리·프레임워크·빌드 도구 **사용 안 함**
- **단일 `index.html` 파일** 하나로 시작 (HTML/CSS/JS 모두 포함). 코드가 커지면 그때 분리 논의.
- 게임 루프는 `requestAnimationFrame` 기반, **델타타임(dt)** 적용으로 프레임 독립적으로 동작.

---

## 4. 핵심 게임 규칙

| 요소 | 내용 |
|---|---|
| 달리기 | 캐릭터는 화면 왼쪽 고정 위치에서 제자리 달리고, **배경/지형/오브젝트가 왼쪽으로 스크롤**된다. |
| 점프 | 키보드 `스페이스`/`↑` 또는 화면 탭 → 점프. 공중에서 한 번 더 입력 시 **더블 점프**. |
| 슬라이드 | 키보드 `↓` 또는 화면 하단 스와이프/버튼 → 슬라이드(자세 낮춤). 누르는 동안 유지. |
| 장애물 | ① 바닥의 가시/장애물(점프로 회피) ② 공중 장애물(슬라이드로 회피) ③ 구덩이/낭떠러지(점프로 통과). |
| 아이템(젤리) | 먹으면 점수 +. 공중·지상에 라인 형태로 배치. |
| 코인/크리스탈 | 별도 재화 카운트(점수와 분리). |
| 체력(HP) | 장애물 충돌 또는 구덩이에 빠지면 HP 감소. HP 0 → 게임 오버. |
| 점수 | 달린 거리 + 먹은 젤리. 시간이 지날수록 점수 자동 증가. |
| 난이도 | 시간이 지날수록 스크롤 속도 점진적 증가, 장애물 빈도 증가. |
| 게임 오버 | 최종 점수 표시 + 재시작 버튼. |

## 5. 화면 구성 (UI 레이아웃)

```
┌─────────────────────────────────────────────┐
│  ❤️ HP바        🪙 코인: 0      ⭐ 점수: 0     │  ← 상단 HUD
│                                               │
│                                               │
│        (하늘/배경 - 패럴랙스 스크롤)            │
│                                               │
│   🍪→        ▲장애물      ●●● 젤리            │  ← 게임 플레이 영역
│  ─────────────────────────────────────────   │  ← 지면(바닥)
└─────────────────────────────────────────────┘
```

- 캐릭터는 화면 **왼쪽 1/4 지점**에 고정.
- 상단 HUD: 좌측 HP 바, 중앙/우측에 코인·점수.
- 모바일에서는 화면 전체가 점프 버튼, 별도 슬라이드 버튼 또는 하단 스와이프.

## 6. 개발 단계 (Phase별 — 순서대로 진행)

### ✅ Phase 1 — MVP (가장 먼저, 여기까지만 우선 만들기)
**목표: "달리고, 점프하고, 장애물에 부딪히면 게임 오버"가 되는 최소 플레이.**

- [ ] Canvas 화면과 게임 루프(`requestAnimationFrame` + dt) 세팅
- [ ] 지면(바닥 라인) 그리기, 캐릭터를 사각형 또는 🍪 이모지로 표시
- [ ] 캐릭터 점프(중력 적용) + 착지 구현
- [ ] 장애물(사각형) 1종류가 오른쪽에서 왼쪽으로 등장
- [ ] 캐릭터-장애물 충돌 판정(AABB 사각형 충돌)
- [ ] 충돌 시 게임 오버 → "GAME OVER + 점수" 표시 + 재시작
- [ ] 시간/거리 기반 점수 카운트 + 상단 표시

> **그래픽은 전부 단색 도형으로.** 이미지·애니메이션 없음. 돌아가는 것만 확인.

### ✅ Phase 2 — 조작·장애물 다양화
- [ ] **더블 점프** 추가
- [ ] **슬라이드** 추가 (누르는 동안 캐릭터 높이 낮아짐)
- [ ] 장애물 종류 분리: 바닥 장애물(점프 회피) / 공중 장애물(슬라이드 회피) / 구덩이(점프 통과)
- [ ] 키보드(스페이스/↑/↓) + 모바일 터치 조작 둘 다 지원

### ✅ Phase 3 — 아이템·체력·난이도
- [ ] 젤리 아이템 등장 + 먹으면 점수 증가 (라인 배치)
- [ ] 코인/크리스탈 재화 별도 카운트
- [ ] HP 시스템: 장애물 충돌 시 HP 감소(즉사 아님), HP 0이면 게임 오버
- [ ] 시간 경과에 따른 **속도/장애물 빈도 점진 상승**

### ✅ Phase 4 — 연출·완성도
- [ ] 패럴랙스 배경(원경/근경 다른 속도로 스크롤)
- [ ] 점프/충돌/획득 효과음 또는 간단한 파티클(반짝임)
- [ ] **보너스 타임**: 젤리를 연속으로 많이 먹으면 게이지가 차고, 발동 시 일정 시간 점수 2배 + 젤리 대량 등장
- [ ] 캐릭터 달리기/점프 애니메이션(스프라이트 또는 간단한 형태 변화)
- [ ] 최고 점수 기록(메모리상 변수로만; 브라우저 저장소는 사용하지 말 것)

> ⚠️ **주의**: 이 게임은 클로드 아티팩트 환경에서는 `localStorage`/`sessionStorage`가 동작하지 않습니다. 최고 점수는 일단 변수로만 관리하고, 영구 저장이 필요하면 별도 환경에서 진행하세요.

## 7. 상세 사양 (시작 수치 — CONFIG로 관리, 자유롭게 튜닝)

```
CONFIG = {
  gravity: 0.8,            // 중력 (값 클수록 빨리 떨어짐)
  jumpPower: -15,          // 점프 초기 속도(음수 = 위로)
  doubleJumpPower: -13,    // 더블 점프 속도
  baseScrollSpeed: 6,      // 시작 스크롤 속도(px/frame 기준)
  speedRampPerSec: 0.05,   // 초당 속도 증가량
  groundY: 화면높이 * 0.8,  // 지면 높이
  playerX: 화면너비 * 0.25, // 캐릭터 고정 x 위치
  maxHP: 3,                // 최대 체력
  obstacleMinGap: 350,     // 장애물 최소 간격(px)
  jellyScore: 10,          // 젤리 1개 점수
  distanceScorePerSec: 50  // 거리 점수 증가율
}
```

- **충돌 판정**: 단순 AABB(사각형) 충돌로 시작. 너무 빡빡하면 히트박스를 캐릭터 실제 크기보다 약간 작게.
- **스폰 규칙**: 일정 거리마다 장애물/젤리를 랜덤하게 생성하되, **점프로 도저히 못 피하는 배치는 금지**(연속 장애물 간 최소 간격 보장).

---

## 8. 게임 상태 (State)

- `READY`: 시작 화면("탭/스페이스로 시작")
- `PLAYING`: 게임 진행 중
- `GAMEOVER`: 게임 오버 화면(점수 표시 + 재시작)
- 상태 전환은 명확한 함수(`startGame()`, `gameOver()`, `resetGame()`)로 관리.

---

## 9. 코드 구조 권장

- `CONFIG` (수치 모음) → `state`(게임 상태/점수/HP) → `player` 객체 → `obstacles[]`, `jellies[]` 배열
- 함수 분리: `update(dt)`, `render()`, `spawnObstacles()`, `checkCollisions()`, `handleInput()`
- 게임 루프: `loop(timestamp)` 안에서 `update` → `render` → `requestAnimationFrame(loop)`

---

## 10. 단계별 테스트 체크리스트

각 Phase 완료 시 아래를 직접 브라우저에서 확인:

- **Phase 1**: 캐릭터가 점프하고 다시 착지하는가? 장애물에 부딪히면 게임 오버되고 재시작되는가? 점수가 올라가는가?
- **Phase 2**: 더블 점프가 되는가? 슬라이드로 공중 장애물을 피할 수 있는가? 키보드·터치 모두 동작하는가?
- **Phase 3**: 젤리를 먹으면 점수가 오르는가? HP가 닳고 0이 되면 끝나는가? 시간이 지날수록 빨라지는가?
- **Phase 4**: 배경이 입체적으로 움직이는가? 보너스 타임이 발동하는가? 전체적으로 "달리는 맛"이 나는가?

---

## 11. 그래픽 / 에셋 방침

- **1차**: 모든 오브젝트를 단색 사각형/원 또는 이모지(🍪 🍬 ⭐ 🔥)로 표시.
- **2차**: 게임이 완전히 돌아간 뒤, 직접 그린 이미지나 무료 에셋으로 교체.
- 코드에서 그리기 부분(`render`)을 따로 분리해 두어 나중에 이미지 교체가 쉽도록 설계.

---

## 12. 클로드코드에 전달할 첫 프롬프트 (복사해서 사용)

> 쿠키런 클래식 스타일의 가로 스크롤 엔드리스 러너 게임을 HTML 단일 파일(index.html)로 만들어줘.
> HTML5 Canvas + 순수 JavaScript만 사용하고 외부 라이브러리는 쓰지 마.
> **이번엔 Phase 1(MVP)만 구현해줘.** 즉:
> 1) 게임 루프(requestAnimationFrame + 델타타임) 2) 지면과 캐릭터(사각형/이모지) 3) 중력+점프+착지 4) 오른쪽에서 오는 장애물 1종 5) 사각형 충돌 판정 6) 충돌 시 GAME OVER + 점수 표시 + 재시작 7) 거리 기반 점수.
> 그래픽은 전부 단색 도형으로 단순하게. 모든 수치는 파일 상단 CONFIG 객체로 모아줘.
> Phase 2 이후 기능(더블점프·슬라이드·아이템·HP·보너스타임)은 아직 넣지 마.

이게 정상 동작하면 "이제 Phase 2 진행해줘"처럼 단계별로 이어가면 됩니다.
